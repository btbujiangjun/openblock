## 1. 方块益智游戏机制

### 核心规则（Block Puzzle 品类）

方块益智游戏（Block Puzzle / 1010! / Block Blast 等）的核心机制：

1. **盘面**：N×N 网格（OpenBlock 默认 8×8）
2. **方块**：每轮提供 K 个（默认 3 个）不同形状的多连块（Polyomino）
3. **放置**：玩家将方块拖拽到合法位置（不超边界，不重叠已有方块）
4. **消除**：整行或整列被填满时自动消除，获得积分
5. **终局**：当所有方块均无合法位置时游戏结束

### 积分规则

| 消除方式 | 基础分 | 备注 |
|---------|--------|------|
| 单行/列 | 10 × 格数 | 一行 8 格得 80 分 |
| 多行同消 | 额外倍率 | 2行=1.2x，3行=1.5x |
| Combo | 叠加系数 | 连续消行获得加成 |

### 策略深度

尽管规则简单，方块益智游戏具有非平凡的策略深度：

- **预期规划**：当前轮 3 块需整体规划，不能只看单块最优
- **空间管理**：避免产生无法填充的小空洞（隔离死区）
- **消行节奏**：堆积到近满时集中消行往往得分更高
- **形状契合**：不同方块间的正负形契合是进阶技巧

### 复杂度分析

- 状态空间：约 2^64（8×8 盘面占用状态），实际约 10^12 有效局面
- 分支因子：平均 ~30（每块约 10 种放置 × 3 块先后顺序）
- 最大博弈深度：约 30-60 步（一局典型时长）

## 2. 游戏难度与心流理论

### 心流理论（Flow Theory）

心流（Flow）由心理学家 Csikszentmihalyi 提出，指人完全投入某活动时达到的最优体验状态：

```
                挑战
                 ↑
                 │     ╱ 心流通道
            焦虑 │   ╱
                 │ ╱
    ─────────────┼─────────────→ 技能
               ╱ │
         无聊 ╱  │
```

**心流通道**：当挑战 ≈ 技能时，玩家进入心流；挑战>>技能→焦虑；技能>>挑战→无聊。

### 在方块游戏中的表现

| 状态 | 游戏表现 | 玩家行为 |
|------|---------|---------|
| **心流** | 清行流畅，不断小目标完成 | 持续游戏，低挫败感 |
| **无聊** | 频繁连击，感觉无挑战 | 降低专注，可能退出 |
| **焦虑** | 频繁near-miss，难以消行 | 高挫败感，可能放弃 |

### 近失效应（Near-Miss Effect）

Near-Miss（近失）：玩家差一点就能消行时的心理状态。

- **心理机制**：大脑将「差一点」解读为「再来一次就能成功」，触发继续游戏动机
- **测量方式**：放置后仍未消行的行/列已填满度 ≥ 7/8
- **商业意义**：Near-Miss 状态是激励广告的最佳触发窗口

## 3. 玩家能力评估模型

### 能力维度定义

OpenBlock 的 `PlayerProfile` 跟踪以下实时维度：

```
技能（Skill）
  = 消行率 × 0.5 + 每步得分 × 0.3 + 生存局数 × 0.2
  ∈ [0, 1]
  含义：玩家在空间规划和得分效率上的综合能力

心流（Flow State）
  = 映射(挫败级别, 连击频率, 节奏平稳度) → {bored, flow, anxious}
  含义：玩家当前的心理投入状态

挫败感（Frustration Level）
  = 连续无消行次数（no_clear streak）
  ∈ [0, ∞)，通常关注 ≥ 4 的区间
  含义：玩家负面情绪积累程度

近失（Near-Miss）
  = 布尔值：本轮放置后是否存在已填 ≥ 7/8 的行/列
  含义：玩家是否处于「差一点」的心理状态

节奏（Pacing）
  = 操作间隔时间的变异系数
  含义：玩家操作节奏的稳定性（低变异=稳定节奏=心流迹象）
```

### 评估的局限性

- **时序依赖**：当前实现基于局内短窗口（~20步），跨局趋势需历史数据
- **置信度门控**：前几步数据量不足，信号可靠性低，通过 `confidenceGate` 缩减调节幅度
- **个体差异**：同一数值对不同玩家意义不同，个性化模型需历史积累

## 4. 自适应出块理论

### 动态难度调节（DDA）

动态难度调节（Dynamic Difficulty Adjustment）是一种根据玩家实时表现自动调整游戏难度的技术。

**目标**：维持玩家在心流通道内，最大化参与时长和愉悦感。

**核心问题**：
1. **测量**：如何量化玩家当前的「挑战-技能」平衡？
2. **调节**：通过什么机制影响难度（不能让玩家察觉）？
3. **延迟**：实时信号 vs 历史趋势如何权衡？

### OpenBlock 的解法

OpenBlock 采用**出块形状分布调节**作为难度控制杆：

- 优点：对玩家透明（不涉及规则变化），调节粒度细腻
- 原理：困难形状（长条、五连块）更难放置，且消行价值更高但时机要求严苛

**10 信号融合**解决了单一指标的局限：

```
单一指标（如分数压力）→ 噪声大，无法区分技巧差异
多信号融合 → 鲁棒性强，覆盖不同维度的玩家状态
```

### 调节的「隐蔽性」原则

优秀的 DDA 应让玩家感受到「这游戏真好玩」而不是「游戏在帮我」：

1. **调节幅度**：单次调节 stress 变化 ≤ 0.05，避免突兀转变
2. **随机性保留**：即使 stress 高，仍保留低 stress 形状一定概率（防止「作弊感」）
3. **玩家主控感**：心流/挫败调节基于玩家行为，而非系统强制
4. **无记忆效应**：每步出块独立计算，避免玩家规律利用

## 5. 玩家心理模型

### 用户分群的心理基础

方块益智游戏玩家可以从两个维度分类：

```
           付费意愿
                ↑
           Whale│  Whale Hardcore
                │  （高技能高付费）
       Dolphin ─┼─
                │  Minnow Casual
                │  （低技能低付费）
    ────────────┼──────────────→ 技能水平
```

| 细分 | 典型行为 | 商业化偏好 |
|------|---------|----------|
| **Whale** | 高频高分，有强竞争欲 | 订阅/大额 IAP，抗广告 |
| **Dolphin** | 中频中分，偶尔追求挑战 | 低价 IAP + 激励广告 |
| **Minnow** | 低频低分，休闲杀时间 | 免费为主，偶尔去广告 |

### 奖励机制与多巴胺循环

游戏留存的核心是**变化比例奖励（Variable Ratio Reward）**：

- **固定奖励**（每步）：放置方块的即时反馈（点击感/音效）
- **短期奖励**（每消一行）：清行动画 + 得分 → ~30秒一次
- **中期奖励**（任务/成就）：每日任务完成 → ~10分钟一次
- **长期奖励**（赛季奖励）：皮肤解锁 → ~1周一次

**关键心理原理**：
- 变比奖励（不确定何时消行）比固定比例奖励产生更强的强化效应
- Near-Miss 效应强化「再试一次」动机
- 成就系统提供「进步感」，对抗技能不足导致的挫败

### 流失风险点

| 场景 | 流失原因 | 对策 |
|------|---------|------|
| 新手（前 5 局） | 规则不清，连续失败 | 简单形状引导，降低 stress |
| 成长期（5-50 局） | 感觉无提升，平台期 | 技能可视化，小目标 |
| 熟练期（50+ 局） | 无聊，缺乏新鲜感 | 赛季内容，挑战模式，排行榜 |
| 高挫败时 | 连续近失但无法消行 | 挫败救济（降压+激励广告） |

## 6. 商业化原理与用户分群

### IAA vs IAP

| 模式 | 全称 | 收入来源 | 适合场景 |
|------|------|---------|---------|
| **IAA** | In-App Advertising | 广告曝光 | 免费游戏，大用户基数 |
| **IAP** | In-App Purchase | 虚拟商品销售 | 有强烈收集/竞争欲的玩家 |

**混合模式**（OpenBlock 策略）：
- Minnow → IAA 为主（激励广告 + 偶尔插屏）
- Dolphin → IAA + 低价 IAP（周卡/小礼包）
- Whale → IAP 为主（月卡/大礼包），尽量减少广告干扰

### 激励广告（Rewarded Ads）的心理设计

激励广告是玩家**主动选择**观看以换取奖励的广告形式：

- **关键**：奖励必须有即时价值（续关、提示、复活）
- **触发时机**：Near-Miss 后（「再来一次」欲望最强）
- **心理接受度**：主动选择 vs 被动插屏 → 满意度显著更高
- **注意限制**：每局上限（OpenBlock: 3次）避免「广告游戏」感

### LTV（用户生命周期价值）

```
LTV = ARPDAU × 留存天数

其中：
  ARPDAU = 广告ARPDAU + IAP ARPDAU
  留存天数 = 受 DDA、任务、社交等系统影响
```

提升 LTV 的两条路径：
1. **提升 ARPDAU**：优化广告触发策略 + 提升 IAP 转化率
2. **提升留存**：DDA 维持心流 + 日常任务 + 赛季内容

### 数据分析指标

| 指标 | 含义 | 健康范围（参考） |
|------|------|----------------|
| Day 1 留存 | 次日回流率 | >40% |
| Day 7 留存 | 7日回流率 | >15% |
| 会话时长 | 平均游戏时长 | 8-15 分钟 |
| 人均日局数 | 每日游戏场次 | 5-15 局 |
| 广告观看率 | 触发后实际观看比例 | >60% |
| IAP 转化率 | 付费用户比例 | 1-5% |

## 7. 强化学习基础

### 核心概念

强化学习（Reinforcement Learning, RL）通过与环境交互来学习最优策略：

```
                  观测 s_t
    环境（游戏）────────────→ Agent（策略网络）
         ↑                          │
    执行动作 a_t                    │ 选择动作 a_t
         └──────────────────────────┘
                  奖励 r_t
```

**核心要素**：
- **状态** s：当前盘面 + 待选方块 + 统计信息
- **动作** a：选择哪个方块放在哪个位置
- **奖励** r：消行得分 + 势函数 + 终局奖惩
- **策略** π(a|s)：给定状态 s 选择动作 a 的概率

### REINFORCE 算法

OpenBlock 浏览器端使用 REINFORCE（策略梯度）算法：

```
目标：最大化期望累计奖励 J(θ) = E[Σ γ^t r_t]

梯度：∇θ J = E[Σ ∇θ log π_θ(a_t|s_t) × (G_t - b_t)]

其中：
  G_t = 从 t 步开始的累计折扣奖励（回报）
  b_t = 价值基线（减少方差）
  γ   = 折扣因子（0.99）
```

### Actor-Critic 架构

PyTorch 端使用 Actor-Critic（残差双塔网络）：

```
输入特征 ψ(s)
    │
    ├──→ 策略头 π(a|s) → 动作概率分布
    │    （Actor：学习什么动作好）
    │
    └──→ 价值头 V(s)   → 局面价值估计
         （Critic：评估当前局面好坏）

损失 = 策略梯度损失 + value_coef × 价值损失 - entropy_coef × 策略熵
```

策略熵（Entropy）防止策略过早收敛到次优：初期高熵探索，后期熵衰减至精细决策。

## 8. RL 在游戏中的应用

### 自博弈（Self-Play）

OpenBlock 的 RL 训练采用**自博弈**模式：Agent 与游戏环境（而非对手）交互，通过最大化得分学习策略。

与围棋/棋类自博弈不同，方块益智游戏是**单人优化**问题，没有对手，但有随机性（出块分布）。

### 课程学习（Curriculum Learning）

为什么需要课程学习？

```
直接训练（随机出块）→ 早期 Agent 几乎无法消行 → 稀疏奖励 → 学习极慢

课程设计：
  阶段 1（0-10k局）：出块偏简单 → 胜利门槛 40 分
  阶段 2（10-40k局）：逐步提高 → 胜利门槛线性升至 220 分
  阶段 3（40k+局）：标准难度 → 维持最终门槛

效果：策略在简单环境中建立基础技能，再迁移到复杂场景
```

### SpawnTransformer（出块模型）

`spawnModel.js` / `rl_pytorch/spawn_model/` 中的 SpawnTransformerV2 是一个辅助模型：

- **训练数据**：人类玩家的游戏行为序列（`move_sequences` 表）
- **功能**：预测「人类偏好的」出块组合，作为自适应出块引擎的额外信号
- **使用方式**：在 Layer 0 为 `blockSpawn` 提供软约束，不替代随机性

### RL 训练的已知挑战

| 挑战 | 表现 | 当前解法 |
|------|------|---------|
| **奖励稀疏** | 早期几乎得不到消行奖励 | 势函数奖励塑形 |
| **动作空间大** | 3块 × ~30位置 ≈ 90 动作 | 掩码非法动作 |
| **随机性** | 出块随机导致方差大 | 价值基线 + 归一化优势 |
| **数值不稳定** | 训练后期梯度爆炸 | 梯度裁剪 + Huber loss |
| **泛化** | 对特定盘面过拟合 | 课程学习 + 随机出块 |
