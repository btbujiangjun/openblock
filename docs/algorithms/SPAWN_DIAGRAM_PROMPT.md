> **已生成图片**：当前版本架构图已保存在
> [`docs/algorithms/assets/spawn-architecture.png`](./assets/spawn-architecture.png)，

> 产出物为 **PNG 格式图片**，视觉规范参考图1（OpenBlock 算法架构总览图）

>
> **使用方式**：复制 §"Prompt 全文"整段粘贴给支持图片输出的大模型

> 发生变化时，同步更新 §"事实包"中对应内容；行号变化不要求严格同步，但
> 模块名、阈值、默认值必须与 `web/src/bot/blockSpawn.js` 代码保持一致。

## 适用场景

- 技术文档配图：为 [`ADAPTIVE_SPAWN.md`](./ADAPTIVE_SPAWN.md) §10.8 生成对应可视化
- 算法评审材料：展示出块从信号感知到染色的完整数据链路
- 新成员 onboarding：给不读源码的协作者提供算法全景图
- 派生：只截取"盘面感知 + 评分构建"等部分子链路的独立图

## 设计原则

1. **语义优先**：图中所有节点使用语义化中文描述，不暴露原始代码标识符，
   让不读源码的读者也能理解每一层的职责。
2. **八层流水线**：体现从"三路输入"到"三块输出+染色"的完整 8 层流水，
   对应 ADAPTIVE_SPAWN.md §10.8 流水线结构。
3. **同花顺三层不变式可读**：识别始终计算、节流在激活门控、单轮预算守卫
   三个设计点必须在图中可以找到对应节点或标注。
4. **视觉层级清晰**：每层独立配色，层间箭头标注主数据流向；
   关键数值（如最多22次重试、填充率52%阈值）写在节点第二行。

## Prompt 全文

````markdown
# 角色

你是一位资深游戏算法架构师，擅长把多阶段评分-抽样系统拆解为清晰的分层
数据流架构图。你必须严格基于下方"流水线描述"中给出的阶段与语义，
**不得发明不存在的阶段，不得把辅助注入画成主路径**。

# 任务

为开源项目 **OpenBlock** 的出块决策引擎（v1.60.35）生成一张
**PNG 格式的视觉架构图**，图名为**「OpenBlock 出块算法架构图」**，
完整呈现从"三路输入"到"三块输出+染色"的 8 层分层数据流。
视觉设计见 §2，流水线内容见 §1，输出要求见 §3。

# §1 八层流水线语义描述

> 算法每轮产出 **3 个候选方块**，流水线从上到下依次经过以下 8 层：

---

## 层 0 · 输入层

三路输入汇入算法入口：

- **游戏棋盘**：当前盘面状态，提供落子合法性、填充情况、行列分析能力
- **策略配置**：包含形状权重、玩家画像（技能/动量/挫败感/会话阶段）
- **出块调度参数**：来自自适应压力引擎的 20+ 调度信号，包括
  消行优先档位（0~3）、多消加权强度、爽感奖励强度、清屏优先强度（峰值×45倍）、
  同花顺目标强度、节奏相位（铺垫期/兑现期）、临消行机会系数、
  空间压力目标、即时兑现目标、新奇形状偏好、个人最佳距离段激活标志

---

## 层 1 · 盘面感知层

对棋盘进行全量感知，产出本轮关键盘面信号：

- **基础状态**：盘面填充率（0~1）、临满行数、空洞数量、表面平整度
- **清屏机会**：清屏准备度（0=无/1=铺垫中/2=即将清屏）；填充率低于45%时关闭
- **同花顺信号**：扫描近满同色行列，分为兑现期（差1~2格即满）和建设期（累积中）
- **同花彩蛋决策**：根据兑现期/建设期行列数动态调整激活概率（3.3%~10%），
  伯努利采样决定本轮是否激活；若激活则将方向匹配的1格×2格/2格×1格细长条加入候选放行名单

---

## 层 2 · 评分构建层

对全部可放置形状进行**真模拟打分**，每个形状独立计算 9 项指标：

| 指标语义 | 说明 |
|---|---|
| 缺口填充数 | 能填入当前盘面缺口的格数 |
| 合法落点数 | 在棋盘上所有合法放置位置数量 |
| 多消潜力 | 放下后能同时消除的行列数（最高3+） |
| 清屏潜力 | 0=无贡献 / 1=铺垫清屏 / 2=直接触发清屏 |
| 精确卡入系数 | 几何完美嵌入邻居/边界的程度（0~1，≥0.999为完美） |
| 同花顺完成潜力 | 能补满某同色行列的格数（**始终计算，不受彩蛋激活影响**） |
| 同花顺目标索引 | 该形状对应的目标同色行/列编号（供染色层使用） |
| 同花顺建设期潜力 | 能为尚未近满的同色行列增加进度的贡献度 |
| 空洞减少潜力 | 填充率超过50%时计算，能消除的隐藏空洞数 |

评分池按**清屏潜力 > 多消潜力 > 缺口填充数**降序排列。

---

## 层 3 · 优先选拔层（消行席位）

从评分池中筛选"消行/爽感类"形状，按优先级填入 1~3 个消行席位：

- **候选池筛选**：缺口填充数 > 0 或 多消潜力 ≥ 1 或 清屏潜力 = 2 或 同花顺完成潜力 ≥ 1
- **席位数量**：清屏准备度≥2 或 清屏优先强度≥0.9 时强制 3 席；否则 1~2 席
- **选拔优先级**（顺序）：直接清屏 → 同花彩蛋激活且同花预算未满 → 多消≥2行 → 随机
- **同花预算守卫**：单轮最多 1 个同花顺块，超额后自动降级为普通消行块

---

## 层 4 · 加权补齐层

对消行席位未填满的剩余槽位，用 **14 维权重乘法链**加权抽样补齐：

| 权重因子 | 触发条件 | 峰值倍率 |
|---|---|---|
| 机动性保障 | 填充率 > 52% 时按合法落点对数加权 | 动态 |
| 直接清屏 | 清屏潜力 = 2 | ×45（清屏强度满时） |
| 清屏铺垫 | 清屏准备度≥1 且有缺口填充 | ×15 |
| 多消潜力 | 多消≥1行 | ×2.7（3行时） |
| 同时多线 | 多线目标≥2 且多消≥2 | ×1.73 |
| 临消行机会 | 临满行数>0 且有缺口填充 | ×3 |
| 精确卡入 | 精确卡入系数≥0.999 | ×2.45 |
| 同花顺完成 | 同花顺潜力≥1 且本轮预算未满 | ×2.0 |
| 同花顺建设 | 建设期潜力 > 0 | ×1.25 |
| 空洞修复 | 填充率>50% 且空洞减少潜力>0 | ×1.4 |
| 体积偏好 | 填充率>52% 且已选块较大 | 小块加权 |
| 个人最佳距离段 | 距个人最佳差距激活时 | ×1.5~2.5 |
| 节奏兑现 | 节奏相位=兑现期 且有缺口填充 | ×1.28 |
| 品类多样性 | 已选品类重复时 | 降权 |

候选池增量维护：每抽一块后仅移除已选形状，同花排除标志首次触发时一次性清理同花候选。
若候选池耗尽则兜底补齐（标记为"兜底块"，三项核心指标均为0）。

---

## 层 5 · 约束验证层（循环最多 22 次）

对选出的三块候选组逐一施加硬约束，不满足则整轮重新抽取（最多 22 次）：

| 约束类型 | 触发条件 | 机制 |
|---|---|---|
| 最低机动性 | 任一块合法落点数 < 当前机动性目标 | 直接淘汰 |
| 顺序可解性 | 盘面填充率 ≥ 52% | 深度优先搜索验证三块可依次放入（预算14000节点；危险区翻倍） |
| 解法空间 | 填充率 ≥ 激活阈值 | 评估三块的解法数量与质量（预算8000节点） |
| 最小空洞增量 | 解法配置窗口内 | 早期尝试中硬过滤产生过多新空洞的候选 |
| 九维区间软过滤 | 早期尝试（前60%）中 | 解法数/空洞增量/终态填充/平整度/危险列等9项区间判定 |

---

## 层 6 · 注入优化层

约束验证通过后，尝试替换弱槽以提升组合质量（特殊形状注入）：

- **救援/压力注入**：按 8 条优先级规则，将救援型或压力型特殊形状替换最弱槽
- **品类多样化注入**：若三块品类高度重复，尝试替换为不同品类形状

---

## 层 7 · 输出层

产出三块候选组及完整决策元数据：

- **三块候选组**：最终出块结果
- **选择元数据**（每块携带）：选择原因、主驱动标签、清屏潜力、同花顺潜力、多消潜力、精确卡入、合法落点数、同花顺目标行列索引
- **选择原因枚举**：直接清屏 / 送同花顺 / 消行 / 综合加权 / 兜底块
- **诊断快照**：三层信号值、已选块明细、解法指标、尝试次数等（供决策流可视化使用）

---

## 底部 · 染色层（下游）

由游戏主逻辑（非出块算法本体）完成颜色绑定：

- **同花顺锁色**：若某块携带同花顺目标行列索引，则强制将该块颜色绑定到对应行列的主色
- **三色无放回抽样**：其余块从当前皮肤色板中不重复抽取颜色
- **决策流可视化**：将选择元数据渲染为可读徽章（直接消行 / ★送清屏 / ★送同花 / 综合选 / 兜底块）

---

## 同花顺设计三层不变式

```
识别层（始终计算）  →  评分构建层中同花顺潜力对所有形状真实计算，不受彩蛋激活影响
节流层（激活门控）  →  彩蛋激活标志（3.3%~10%概率）控制是否选入；
                        优先选拔层和加权补齐层均设有同花候选过滤
预算守卫（双修护）  →  单轮最多1个同花块；选拔层与补齐层对称检查（v1.60.35修复）
染色绑定（闭环）    →  同花顺目标行列索引随选择元数据下传至染色层强制锁色
```

# §3 输出要求

直接输出 **PNG 格式图片**，实现方式由你自主决定。要求：

1. 图片尺寸 1400×1000px 或同比例更高分辨率
2. 节点标签使用 §1 中的语义名称，不使用原始代码标识符
3. 每层节点数 5~9 个，横向均匀排布
4. 图片清晰可读，文字无截断、无溢出

# §4 自检清单（输出前对照）

- [ ] 8 个主层（含染色底层）全部存在且配色各异

- [ ] 染色层体现同花顺锁色和无放回三色抽样
- [ ] 右侧信息栏含同花顺三层不变式
- [ ] 所有节点使用语义中文标签，无原始代码标识符
- [ ] 输出为 PNG 图片

现在请基于以上流水线描述，输出 PNG 格式的出块算法架构图。
````
