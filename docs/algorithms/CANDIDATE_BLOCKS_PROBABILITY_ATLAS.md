# 候选块概率图鉴

> **来源**：由 Canvas `candidate-blocks.canvas.tsx` 转换。  
> **定位**：说明 OpenBlock 候选块池、基础抽样概率、难度档位权重和运行时动态因子，便于验证“出块概率”展示与策略逻辑。

---

## 1. 核心结论

OpenBlock 的候选块从 **28 个形状** 池中按类别权重抽样。基础概率由：

> **基础概率 = 类别权重 / Σ(类别权重 × 类别成员数)**

运行时再叠加清屏、多消、机动性、节奏、玩家画像等 12+ 个动态因子，形成最终选择。

| 指标 | 数值 |
|------|------|
| 候选形状总数 | 28 |
| 形状类别 | 7 |
| 平均单元数 | 约 3.7 |
| 自适应难度档位 | 10 |

---

## 2. Normal 难度类别占比

单格抽样中各类别的概率分布由权重和成员数共同决定。L 形池有 8 个成员，权重 1.20 时合计占比最高，但每个具体 L 形会被成员数稀释。

| 类别 | 权重 w | 成员数 n | w × n | 概率含义 |
|------|--------|----------|-------|----------|
| 线条 | 1.40 | 4 | 5.60 | 一维延展块，更适合填长缝和消线 |
| 矩形 | 1.20 | 2 | 2.40 | 块面积较大，适合低压热身和填补大空区 |
| 方形 | 1.10 | 2 | 2.20 | 稳定填充，风险中等 |
| T 形 | 0.95 | 4 | 3.80 | 需要局部结构匹配，能制造或修复凸凹结构 |
| Z 形 | 0.90 | 4 | 3.60 | 更依赖边缘和错位结构 |
| L 形 | 1.20 | 8 | 9.60 | 成员最多，合计占比高，单形状概率被稀释 |
| J 形 | 0.95 | 4 | 3.80 | 与 L 形互补，补偿镜像结构 |

---

## 3. 单形状概率解释

单形状概率不是简单看类别权重，而是看“类别权重被类别成员数分摊后的结果”。

例如：

- L 形类别整体占比高，是因为成员数多。
- 但单个 L 形的概率低于类别直觉，因为 8 个 L 形成员共享同一个类别权重。
- 线条类成员少，所以单个线条块的概率通常更高。

这也是 UI 中“出块概率”必须展示 **具体形状概率** 而不是只展示类别权重的原因。

---

## 4. 难度权重矩阵的作用

自适应引擎根据玩家压力 `stress ∈ [-0.2, 0.85]` 在多个 profile 间插值出实时权重；菜单难度 Easy / Normal / Hard 提供静态基线。

| 档位类型 | 作用 | 策略倾向 |
|----------|------|----------|
| Easy | 降低操作负担 | 增加小块、线性块和高机动块，减少复杂拐角块 |
| Normal | 默认平衡 | 在可消行、可填缝和多样性之间平衡 |
| Hard | 提升规划要求 | 增加更依赖顺序、位置和空间结构的块 |
| Adaptive Low Stress | 玩家低压 | 允许更大块、更复杂块，制造挑战 |
| Adaptive High Stress | 玩家高压 | 增加可放置、可救援、可消行的块，降低死局概率 |

## 5. 运行时动态因子

| 层级 | 因子 | 倍率 | 物理含义 |
|------|------|------|----------|
| Layer 1 | 完美清屏 | ×12.0 | 该块放置后可直接清空全盘，最高优先级 |
| Layer 1 | 清屏准备 | ×1-7 | 盘面接近清空时，gap 填充块大幅加权 |
| Layer 1 | 多消潜力 | ×1.6-2.7 | `multiClear >= 2` 时指数加权 |
| Layer 1 | 机动性 | ×log(1+P) | 合法落点越多权重越高，填充率越高越敏感 |
| Layer 1 | 空洞修复 | ×1-2 | `fill > 0.5` 且能减少空洞的块 |
| Layer 1 | 临消行加成 | ×1-3 | `nearFullLines × 2.0`，鼓励促清 |
| Layer 2 | Combo 链 | ×1-1.8 | 连击活跃时偏好消行块 |
| Layer 2 | 节奏 payoff | ×1.7 | 收获相位放大 gap 填充块 |
| Layer 2 | Size 偏好 | ×0.5-2.5 | 高填充偏小块，热身偏大块 |
| Layer 2 | 类别多样性 | ×0.2-1 | 同轮重复类别 + 跨轮记忆衰减 |
| Layer 3 | ClearGuarantee | ×1.6-2.1 | 强制保证 N 个消行块或提示块 |
| Layer 3 | 里程碑 | ×1.3 | 刚跨分数门槛时偏好消行块 |

## 6. UI 展示建议

候选块概率展示应遵循：

- 展示“出块概率”，不要展示“候选块概率”这种容易误解的标签。
- 优先展示最终归一化后的具体形状概率。
- 类别权重只作为策略解释，不作为用户主指标。
- 动态因子应独立说明，避免用户把基础概率误认为最终概率。
- 若同一个指标有快照和趋势两种形式，优先保留趋势曲线，快照做紧凑辅助。

## 7. 策略合理性检查

| 检查项 | 合理表现 | 风险表现 |
|--------|----------|----------|
| 高压局 | 小块、可放置块、消行块概率上升 | 仍持续生成大块或低机动块 |
| 低压局 | 大块、复杂块、多样性上升 | 长期只给保守块，导致无聊 |
| 多消窗口 | 可触发多线消除的块被放大 | 多消机会存在但概率不变 |
| 清屏窗口 | gap 填充块被强加权 | 清屏差一步时仍随机出块 |
| 类别重复 | 同类块短期降权 | 连续多轮重复同一结构导致体验单调 |

## 8. 个性化对出块概率的影响

新增 `motivationIntent` 只改变概率倾向，不绕过可解性与几何校验：

| 个性化信号 | 概率变化 | 约束 |
|------------|----------|------|
| 回归暖启动 | 小块、可消行块、多消友好块权重上升 | 只在沉默回归窗口生效 |
| 可访问性负担 | 小块和高机动块上升，复杂顺序块下降 | 关闭 `orderRigor` |
| 收集/完成动机 | 同 icon / bonus 目标上升 | 有近满线或可兑现几何时才强化多消 |
| 高手挑战动机 | 多样性、多消、顺序规划块上升 | 高风险盘面仍优先救济 |
| 社交公平挑战 | 个性化概率关闭 | 固定 seed/固定规则保证可比 |
