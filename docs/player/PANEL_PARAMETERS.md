## 2. 能力指标区

能力指标区每次面板刷新都会用当前棋盘重新构建 `AbilityVector`：`skillScore`、`controlScore`、`clearEfficiency` 主要来自 `PlayerProfile` 滑动窗口；`boardPlanning`、`riskLevel` 会叠加当前 `analyzeBoardTopology(grid)` 的 `fillRatio / holes / closeLines`。因此它与下方实时拓扑指标使用同一盘面口径。

> **v2 (2026-05) 体感升级**：6 个 pill 的 hover tooltip 已扩写到 v2 公式语义；
> - 视觉分组：能力 / 操作 / 消行 / 规划 = 暖色（向上=好），风险 = 红色（向下=好），置信 = 灰色（数据元）；
> - hover 任意 pill 弹出 6 维 SVG 雷达浮层，风险轴在雷达内被翻转为 1−x，让"向外=好"的语义在所有 6 个轴上一致；
> - `controlScore` / `clearEfficiency` 各自使用独立窗口（8 步 / 16 步），不再共用 PlayerProfile 全局窗口。
> - 详细公式与权重见 [`ALGORITHMS_PLAYER_MODEL.md §13.7`](../algorithms/ALGORITHMS_PLAYER_MODEL.md)。

### 2.1 技能 (skillLevel)

| 属性 | 值 |
|------|-----|
| **面板标签** | `技能` |
| **展示格式** | 百分比 (0%~100%) + 进度条 |
| **取值范围** | 0~1 (显示为 0%~100%) |
| **数据源** | `PlayerProfile.skillLevel` |

**数学定义**：

```
rawSkill = thinkScore × 0.15
         + clearScore × 0.30
         + comboScore × 0.20
         + missScore  × 0.20
         + loadScore  × 0.15

skillLevel(t) = skillLevel(t-1) + α × (rawSkill - skillLevel(t-1))
```

其中：
- `thinkScore = 1 - clamp((thinkMs - 800) / 12000, 0, 1)` — 思考越快得分越高
- `clearScore = min(1, clearRate / 0.55)` — 消行率越高得分越高
- `comboScore = min(1, comboRate / 0.45)` — 多行消除比率
- `missScore = 1 - min(1, missRate / 0.3)` — 失误率越低得分越高
- `loadScore = 1 - cognitiveLoad` — 认知负荷越低得分越高
- `α` = 0.35（前 5 步贝叶斯快速收敛）或 0.15（稳态平滑）

**物理含义**：综合反映玩家的操作水平。类似于棋类游戏中的 Elo 评分，但采用指数移动平均而非胜负制。高技能意味着玩家思考迅速、消行效率高、失误少、操作流畅。

**系统影响**：
- skillLevel > 0.5 → 系统加压（出更多不规则块）
- skillLevel < 0.5 → 系统减压（出更多消行友好块）
- 跨局持久化到 localStorage，24 小时未玩会向 0.5 缓慢衰减

### 2.2 消行 (clearRate)

| 属性 | 值 |
|------|-----|
| **面板标签** | `消行` |
| **展示格式** | 百分比 (0%~100%) |
| **取值范围** | 0~1 |
| **数据源** | `PlayerProfile.metrics.clearRate` |

**数学定义**：

```
clearRate = 触发消行的放置次数 / 有效放置总次数
```

统计基于滑动窗口最近 15 步中**非 AFK** 的放置操作。

**物理含义**：衡量玩家的空间规划能力——每放置一个块，有多大概率能凑满整行或整列并触发消除。类比篮球的投篮命中率。

**行为解读**：
- 35%+ → 正常水平，能稳定消行
- 50%+ → 高手，几乎每步都能消行
- <15% → 困难，空间规划能力不足或棋盘状态恶劣

### 2.3 失误 (missRate)

| 属性 | 值 |
|------|-----|
| **面板标签** | `失误` |
| **展示格式** | 百分比 (0%~100%) |
| **取值范围** | 0~1 |
| **数据源** | `PlayerProfile.metrics.missRate` |

**数学定义**：

```
missRate = 放置失败(miss)次数 / 窗口内总操作次数
```

Miss 定义：玩家将块拖拽到不合法位置后释放，块弹回 dock。

**物理含义**：反映玩家的操作精准度和对棋盘空间的感知能力。高失误率可能意味着：
- 手指/鼠标操作不精确
- 对块形与空间的匹配判断有误
- 界面交互体验需优化

**系统影响**：missRate > 28% 会触发 anxious 心流状态，系统开始减压。

### 2.4 思考 (thinkMs)

| 属性 | 值 |
|------|-----|
| **面板标签** | `思考` |
| **展示格式** | 毫秒 (如 `2134ms`) |
| **取值范围** | 0~60000ms（上限截断） |
| **数据源** | `PlayerProfile.metrics.thinkMs` |

**数学定义**：

```
thinkMs = 窗口内非AFK放置操作的平均思考时间
        = Σ(每次放置时间戳 - 上次操作时间戳) / 有效放置数
```

超过 15s 的操作视为 AFK（离开），排除出计算。

**物理含义**：衡量玩家的决策速度，是认知能力的直接体现。类比国际象棋中的平均每步用时。
- **短**（<1.2s）：操作非常快，可能过于简单（bored 候选）
- **中**（1.2s~10s）：正常思考范围
- **长**（>10s）：犹豫不决，可能面对困难局面（anxious 候选）

**系统影响**：直接参与 skillLevel 计算（思考越快，thinkScore 越高）；<1.2s + 高消行 + 低失误 → bored 判定。

### 2.5 负荷 (cognitiveLoad)

| 属性 | 值 |
|------|-----|
| **面板标签** | `负荷` |
| **展示格式** | 百分比 (0%~100%) |
| **取值范围** | 0~1 |
| **数据源** | `PlayerProfile.cognitiveLoad` |

**数学定义**：

```
variance = Σ(thinkMs_i - avg_thinkMs)² / N
cognitiveLoad = min(1, variance / 8,000,000)
```

其中 8,000,000 ms² 为高方差阈值（可配置）。

**物理含义**：衡量玩家的**决策一致性**。如果每步思考时间大致相似，说明局面在能力范围内（低负荷）；如果有些步骤秒放、有些步骤思考很久，说明某些局面让玩家犹豫，认知资源被大量消耗。

这一指标源自认知心理学的「认知负荷理论」（Cognitive Load Theory, Sweller 1988）。在教育游戏设计中，认知负荷过高意味着任务超出了工作记忆容量。

**行为解读**：
- <30% → 操作节奏均匀，游刃有余
- 30%~70% → 部分局面有挑战，正常学习曲线
- \>70% → 频繁遇到困难决策，可能需要减压或提供提示

### 2.6 APM (engagementAPM)

| 属性 | 值 |
|------|-----|
| **面板标签** | `APM` |
| **展示格式** | 小数 (如 `8.2`) |
| **取值范围** | >0 |
| **数据源** | `PlayerProfile.engagementAPM` |

**数学定义**：

```
APM = 窗口内操作次数 / (窗口时间跨度 / 60000)
    = Actions Per Minute
```

**物理含义**：源自 RTS 游戏（如星际争霸）的 APM 概念，衡量玩家的操作频率和参与度。APM 高说明玩家投入度高、操作活跃；APM 骤降可能意味着分心、疲劳或被其他事务打断。

**行为解读**：
- 8~15 → 正常休闲游戏节奏
- <4 → 可能在思考、被打断或即将放弃
- \>20 → 非常活跃，可能处于心流状态

## 3. 实时状态信号区

### 3.1 flowState（心流状态）

| 属性 | 值 |
|------|-----|
| **面板标签** | `● flow` / `▲ bored` / `▼ anxious` |
| **展示格式** | 彩色标签 |
| **取值** | `flow` / `bored` / `anxious` |
| **数据源** | `PlayerProfile.flowState` |

**判定逻辑**：

```
if F(t) < 0.25 → flow (沉浸区，跳过所有其他判定)

if thinkMs < 1200 && clearRate > 45% && missRate < 5% → bored
if missRate > 28% → anxious
if thinkMs > 10000 && clearRate < 15% → anxious
if avgFill > 78% && clearRate < 20% → anxious
if cognitiveLoad > 70% && clearRate < 25% → anxious
if F(t) > 0.5 && clearRate > 40% → bored

else → flow
```

**物理含义**：源自 Csíkszentmihályi 的心流理论（Flow Theory, 1990）。心流是一种完全投入的最优体验状态，发生在挑战与能力匹配时：
- **flow（心流）** ●：挑战 ≈ 能力，玩家沉浸其中，体验最佳
- **bored（无聊）** ▲：挑战 < 能力，玩家觉得太简单 → 系统加压
- **anxious（焦虑）** ▼：挑战 > 能力，玩家觉得太难 → 系统减压

**系统影响**：直接调节出块引擎的 flowAdjust 信号维度，bored 加压 +0.08~+0.16，anxious 减压 -0.12~-0.24。

---

### 3.2 F (flowDeviation)

| 属性 | 值 |
|------|-----|
| **面板标签** | `F 0.18` |
| **展示格式** | 数值 + 颜色（绿/黄/红） |
| **取值范围** | 0~2（通常 0~1） |
| **数据源** | `PlayerProfile.flowDeviation` |

**数学定义**：

```
boardPressure = avgFill × 0.45 + clearDeficit × 0.35 + cognitiveLoad × 0.2
F(t) = |boardPressure / skillLevel − 1|
```

其中：
- `avgFill`：窗口内非 AFK 放置操作的平均棋盘填充率
- `clearDeficit = 1 - min(1, clearRate / 0.4)`：消行率不足程度
- `cognitiveLoad`：认知负荷

**物理含义**：量化「挑战-能力比」与理想值 1.0 的偏离程度。这是对 Csíkszentmihályi 心流理论的数学建模：
- **F(t) < 0.25**（绿色）：沉浸区，挑战与能力平衡
- **F(t) 0.25~0.5**（黄色）：轻度偏移，需要微调
- **F(t) > 0.5**（红色）：显著偏移，需要强干预

**与 flowState 的关系**：F(t) 是连续量化值，flowState 是离散分类。F(t) < 0.25 时直接返回 flow，无需检查其他条件。F(t) 越大，flowAdjust 的修正幅度按 `min(2, 1+F(t))` 放大。

### 3.3 pacingPhase（节奏相位）

| 属性 | 值 |
|------|-----|
| **面板标签** | `tension` / `release` |
| **展示格式** | 灰色标签 |
| **取值** | `tension`（紧张期）/ `release`（松弛期） |
| **数据源** | `PlayerProfile.pacingPhase` |

**数学定义**：

```
pos = spawnCounter mod cycleLength(5)
if pos < tensionPhases(3) → tension
else → release
```

**物理含义**：模拟音乐的「副歌-间奏」结构。游戏设计理论（Schell《The Art of Game Design》）强调节奏张弛对玩家体验的重要性。如果难度只单调递增，玩家很快倦怠。

- **tension（紧张期）**：前 3 轮出块略加压（+0.04），考验玩家空间规划
- **release（松弛期）**：后 2 轮出块略减压（-0.12），给予喘息和成就感

5 轮为一个完整周期，循环往复。

---

### 3.4 sessionPhase（会话阶段）

| 属性 | 值 |
|------|-----|
| **面板标签** | `early` / `peak` / `late` |
| **展示格式** | 灰色标签 |
| **取值** | `early` / `peak` / `late` |
| **数据源** | `PlayerProfile.sessionPhase` |

**判定逻辑**：

```
if spawnCounter ≤ 2 || elapsed < 30s → early
if elapsed < 300s (5分钟) → peak
else → late
```

**物理含义**：反映玩家在当前游戏局中的时间阶段，对应运动心理学中的「热身-巅峰-疲劳」周期：
- **early（热身）**：刚开局，数据不足，策略保守
- **peak（巅峰）**：30s~5min，玩家专注力最强，系统策略最活跃
- **late（疲劳）**：>5min，注意力可能下降，结合动量下降时建议休息

### 3.5 动量 (momentum)

| 属性 | 值 |
|------|-----|
| **面板标签** | `动量 0.12` |
| **展示格式** | 数值 + 颜色（绿/黄/红） |
| **取值范围** | -1~1 |
| **数据源** | `PlayerProfile.momentum` |

**数学定义**：

```
将滑动窗口一分为二：
olderCR = 前半段的消行率
newerCR = 后半段的消行率
momentum = clamp((newerCR - olderCR) / 0.3, -1, 1)
```

**物理含义**：借用物理学「动量」概念，衡量玩家表现的**变化趋势**（一阶导数）：
- **正动量（绿色，>0.15）**：消行率上升趋势，玩家在进步或找到了打法
- **零动量（黄色，-0.15~0.15）**：表现稳定
- **负动量（红色，<-0.15）**：消行率下降趋势，可能遇到瓶颈或疲劳

**系统影响**：
- 正动量 + combo ≥ 2 → comboRewardAdjust 加压（正反馈，保持兴奋）
- 负动量 + late 阶段 → sizePreference 偏小（降低难度）

### 3.6 未消 (frustrationLevel)

| 属性 | 值 |
|------|-----|
| **面板标签** | `未消 3` |
| **展示格式** | 数值 + 颜色（绿/黄/红） |
| **取值范围** | ≥0 (整数) |
| **数据源** | `PlayerProfile.frustrationLevel` |

**数学定义**：

```
每次放置后：
  if cleared → consecutiveNonClears = 0
  else → consecutiveNonClears++
frustrationLevel = consecutiveNonClears
```

**物理含义**：直接衡量玩家的**连续挫败程度**。源自游戏设计中的「挫败感管理」理论——连续多步未消行会让玩家产生"卡住了"的负面感受。

**行为解读与系统响应**：
- **0~1（绿色）**：正常状态
- **2~3（黄色）**：轻度积累，提醒关注
- **≥4（红色）**：触发挫败救济机制：
  - stress 减少 0.18
  - clearGuarantee ≥ 2（出块偏向消行友好）
  - sizePreference = -0.3（偏向小块，便于腾挪）

---

### 3.7 闭环 (feedbackBias)

| 属性 | 值 |
|------|-----|
| **面板标签** | `闭环 +0.012` |
| **展示格式** | 带符号数值 + 颜色 |
| **取值范围** | -0.15~0.15 |
| **数据源** | `PlayerProfile.feedbackBias` |
| **显示条件** | \|feedbackBias\| > 0.005 时才显示 |

**数学定义**：

```
每次出块 (recordSpawn):
  bias *= decay(0.8)              // 历史衰减
  开启观察窗口 horizon=4 步

窗口结束后:
  delta = 实际消行数 - expected(1)
  bias += delta × alpha(0.02)
  bias = clamp(-0.15, +0.15)
```

**物理含义**：这是一个**闭环控制器**（类似工程中的 PID 控制）。系统出了一批块后，观察玩家的实际消行表现：
- 消行多于预期 → bias 正值 → 下次出块可以略加压
- 消行少于预期 → bias 负值 → 下次出块应减压

这避免了纯开环控制（只看画像参数出块，不看效果）的问题。

**行为解读**：
- **正值（绿色，>0.03）**：前几轮出块后消行效果好，系统有余量加压
- **约零（黄色）**：出块与消行匹配，系统稳定
- **负值（红色，<-0.03）**：出块后消行不足，系统需要进一步减压

### 3.8 AFK

| 属性 | 值 |
|------|-----|
| **面板标签** | `AFK 2` |
| **展示格式** | 数值（黄色标签） |
| **取值范围** | ≥0 (整数) |
| **数据源** | `PlayerProfile.metrics.afkCount` |
| **显示条件** | afkCount > 0 时才显示 |

**数学定义**：

```
afkCount = 窗口内 thinkMs ≥ 15000ms 的放置操作数
```

**物理含义**：检测玩家是否在操作期间离开（Away From Keyboard）。15 秒以上的间隔极大概率不是在思考而是在做其他事。这些操作的 thinkMs 会严重拉高均值，干扰技能评估，因此被排除出 metrics 计算。

### 3.9 轮次 (spawnRoundIndex)

| 属性 | 值 |
|------|-----|
| **面板标签** | `轮次 6` |
| **展示格式** | 灰色标签 |
| **取值范围** | ≥0 (整数) |
| **数据源** | `PlayerProfile.spawnRoundIndex` |

**数学定义**：本局内 dock 刷新（出块）的次数。每调用一次 `recordSpawn()` 递增 1，新局开始时重置为 0。

**物理含义**：标记游戏进程，是节奏系统（pacing）的驱动时钟。轮次也用于判定新手保护窗口（前 5 轮）和会话阶段。

### 3.10 条件标签

以下标签仅在特定条件下显示：

| 标签 | 条件 | 物理含义 |
|------|------|---------|
| **⚡ 近失** | `hadRecentNearMiss = true` | 上一步在 >60% 填充的棋盘上放置但未消行。玩家差一点就能消行，这种"近失"状态研究表明（Clark et al. 2009）能强化续玩意愿。系统降压并提高消行保证 |
| **↻ 恢复** | `needsRecovery = true` | 棋盘曾达到 >82% 填充率，触发紧急恢复模式（倒计时 4 步）。系统大幅降压（-0.2），出小块/长条帮助清行 |
| **✦ 新手** | `isInOnboarding = true` | 终身放置 <20 次且当前轮次 ≤5。系统将 stress 钳制到 ≤-0.15，只出规整块型，保护新手建立信心 |

---

## 4. 投放参数区

本节分两类口径：`压力 / F(t) / 闭环 / 目标保消 / 尺寸 / 多样 / shapeWeights / V3 元信息 / 回退原因` 是「上一轮出块」决策瞬间由 `game.js` → `_captureAdaptiveInsight` 和 `_commitSpawn` 捕获的快照；`占用 / 空洞 / 平整 / 近满 / 多消候选 / 清屏候选` 按当前棋盘实时重算，用来与上方能力指标对齐。

出块模式顶栏与侧栏单选保持一致：`启发式` 表示使用 `adaptiveSpawn + blockSpawn`；`生成式` 表示请求 SpawnTransformerV3，但仍会先生成启发式轨兜底与诊断。V3 成功时显示 `modelVersion / personalized / feasibleCount`；V3 输出非法、服务不可用或护栏失败时显示 `fallbackReason`，实际 dock 使用启发式轨。

「目标保消」= **目标保证可消行**档位；「尺寸」= 块格数偏好（非物理「尺子」）；「多样」= 三连品类新鲜度。

### 4.1 stress（综合压力）

| 属性 | 值 |
|------|-----|
| **面板标签** | `压力 0.32`（界面）；下文仍称 stress |
| **取值范围** | -0.2~1.0 |
| **数据源** | `resolveAdaptiveStrategy()._adaptiveStress` |

**数学定义**（详见 [REALTIME_STRATEGY.md](./REALTIME_STRATEGY.md)）：

```
stress = scoreStress + runStreakStress + skillAdjust + flowAdjust
       + pacingAdjust + recoveryAdjust + frustrationRelief
       + comboReward + nearMissAdjust + feedbackBias
```

信号维度包含分数、连战、难度模式、能力风险、心流、节奏、恢复/挫败、combo、近失、闭环反馈、拓扑压力和局间弧线，最终 clamp 到 [-0.2, 1.0]。

**物理含义**：这是出块引擎的**核心控制变量**。stress 决定了系统给玩家出什么形状的块：
- **低 stress（<0.2）**：以线条、矩形为主，消行容易（"简单模式"）
- **中 stress（0.3~0.5）**：各类块型均衡（"标准模式"）
- **高 stress（>0.6）**：T/Z/L/J 不规则块增多，需要更强的空间规划（"困难模式"）

stress 通过线性插值映射到 10 档 shapeWeights profile。

---

### 4.2 F(t)（心流偏移快照）

投放参数区的 F(t) 是**上一次出块时**捕获的心流偏移快照，与状态区的 F 值可能有微小差异（状态区是实时值）。

---

### 4.3 闭环（反馈偏移快照）

同上，是出块决策时刻的 feedbackBias 快照。

---

### 4.4 占用（棋盘填充率）

| 属性 | 值 |
|------|-----|
| **面板标签** | `占用 42%`（界面）；含义为当前盘面占用率 |
| **取值范围** | 0%~100% |
| **数据源** | 当前 `grid.getFillRatio()` |

**数学定义**：

```
fill = 棋盘上已被占据的格子数 / 总格子数(64)
```

**物理含义**：棋盘的拥挤程度。fill 直接参与 boardPressure 的计算（权重 45%），是影响心流判定的最大因子。高填充意味着空间紧张，生存风险增大。

---

### 4.5 目标保消 (clearGuarantee)

| 属性 | 值 |
|------|-----|
| **面板标签** | `目标保消 2`（界面全称为目标消行保证档位） |
| **取值范围** | 1~3 (整数) |
| **数据源** | `spawnHints.clearGuarantee` |

**物理含义**：告诉出块生成器，在 3 个候选块中，至少要有几个块**能与当前棋盘缺口匹配**（放置后可消行）。
- `清1`（默认）：无特殊保证
- `清2`：至少 2 个块有消行潜力（挫败/恢复/近失/新手时触发）
- `清3`：最高保证（极端恢复场景）

---

### 4.6 尺寸 (sizePreference)

| 属性 | 值 |
|------|-----|
| **面板标签** | `尺寸 -0.3`（界面；表示块尺寸偏好，非度量工具） |
| **取值范围** | -1~1 |
| **数据源** | `spawnHints.sizePreference` |

**物理含义**：控制候选块的格数偏好：
- **负值**（如 -0.5）：偏向小块（2~3 格），便于在狭小空间中腾挪，恢复/挫败时使用
- **零**：无偏好
- **正值**：偏向大块（4~5 格），增加挑战或清板效率

---

### 4.7 多样 (diversityBoost)

| 属性 | 值 |
|------|-----|
| **面板标签** | `多样 0.15`（界面；品类新鲜度） |
| **取值范围** | 0~1 |
| **数据源** | `spawnHints.diversityBoost` |

**物理含义**：惩罚连续出现相同品类块的权重。当玩家处于 bored 状态时，系统注入新鲜感——增加 diversityBoost 使得 3 个候选块更倾向不同类型，避免审美疲劳。

---

### 4.8 形状权重（shapeWeights top 5）

| 属性 | 值 |
|------|-----|
| **面板标签** | `长条 2.2  矩形 1.7  方形 1.4  T 形 1.1  Z 形 1.1` |
| **取值范围** | >0 (相对权重) |
| **数据源** | `resolveAdaptiveStrategy().shapeWeights` top 5 |

**物理含义**：7 大块型类别（长条/矩形/方形/T 形/Z 形/L 形/J 形）的**抽样概率权重**。权重越高，该类别被选中的概率越大。

权重由 stress 通过插值从 10 档 profile 中生成。例如在 stress=0 时，长条权重 2.5 是 Z 形权重 0.65 的 3.8 倍；在 stress=0.85 时，Z 形权重 1.48 反超长条的 1.45。

面板只展示 top 5 以保持简洁。
