# Web 前端国际化（i18n）

> 本文描述网页端（`web/`）文案多语言的架构、约定与扩展方式。  
> **代码为准**：以 `web/src/i18n/` 下实现为准；本文随功能迭代更新。

微信小程序端使用轻量 i18n：`miniprogram/core/i18n.js` 内置 `zh-CN` / `en`，并包含 34 套皮肤名翻译。小程序不加载 Web 端 `web/src/i18n/locales/*`，需要单独维护。

---

## 1. 概述

- **形式**：各语言一个 ES 模块，**扁平键值**（`string → string`），默认回退 **简体中文 `zh-CN`**。
- **入口**：`web/src/i18n/i18n.js`（`initI18n`、`t()`、`applyDom()`、`setLocale()` 等）。
- **持久化**：`localStorage` 键 `openblock_locale_v1`，存储当前语言代码（如 `en`、`pt-BR`）。
- **UI**：主菜单底部 `#locale-select` 由 `main.js` 根据 `AVAILABLE_LOCALES` **动态生成选项**（勿与代码列表脱节）。
- **RTL**：`ar`（阿拉伯语）下设置 `document.documentElement.dir = 'rtl'`，其余为 `ltr`。
- **文档中心**：本文档已在服务端注册，可从 [文档中心](/docs) → **开发者指南** 打开。

## 2. 目录结构

```
web/src/i18n/
├── i18n.js              # 运行时：当前语言、t()、applyDom、tSkinName、订阅与 RTL
└── locales/
    ├── zh-CN.js         # 默认字典（缺失键时回退至此）
    ├── en.js
    ├── …                # 其余语言，文件名与语言代码对应（pt-BR → pt-BR.js）
    └── uk.js
```

## 4. 核心 API（`i18n.js`）

| 导出 | 作用 |
|------|------|
| `initI18n()` | 启动时从 `localStorage` 读取并 `setLocale` |
| `getLocale()` / `setLocale(code)` | 读写当前语言；未知代码回退 `zh-CN` |
| `t(key, vars?)` | 取文案；支持占位符 `{{name}}`（由 `vars` 替换） |
| `tSkinName(skin)` | 棋盘主题显示名：键 `skin.name.<skin.id>`，无键时回退 `skins.js` 的 `name` |
| `applyDom(root?)` | 根据 `data-i18n*` 写入 DOM（见下节） |
| `applyMeta()` | 同步 `document.title` 与 `<meta name="description">` |
| `subscribeLocale(cb)` | 语言切换后回调（用于刷新皮肤下拉、`updateUI` 等） |
| `AVAILABLE_LOCALES` | `{ code, nativeName }[]`，供语言下拉与文档列举 |

---

## 5. 文案键约定（节选）

| 前缀 | 用途 |
|------|------|
| `meta.title` / `meta.description` | 页面标题与描述 |
| `ui.stat.*` / `ui.skin.*` / `ui.header.*` / `ui.aria.*` / `ui.skill.*` | 顶栏、统计、无障碍与技能条 |
| `menu.*` | 主菜单按钮与「语言」标签 |
| `dailyMaster.*` | 每日大师题提示与 Toast |
| `game.over.*` / `game.retry` / `game.menu` | 结算与按钮 |
| `progress.rank.*` / `progress.streakDays` / `best.gap` | 等级称号与 streak |
| `skin.name.<id>` | 与 `web/src/skins.js` 中皮肤 `id` 一致（如 `neonCity`） |
| `boot.fileProtocol` | `file://` 打开时的引导文案 |

新增 UI 时：**先在 `zh-CN.js` 增加键**，再同步到其他语言文件（至少英文可先对齐，其余可渐进补齐）。

小程序新增 UI 时，同步更新 `miniprogram/core/i18n.js`：

- 通用文案放在 `MESSAGES[lang]` 顶层键。
- 皮肤名放在 `MESSAGES[lang].skinNames[skinId]`。
- 主菜单皮肤下拉通过 `getSkinListMeta()` 间接读取当前语言下的皮肤名。

## 6. DOM 绑定属性

由 `applyDom()` 处理（切换语言后对根节点重跑即可）：

| 属性 | 行为 |
|------|------|
| `data-i18n="key"` | `textContent = t(key)`；可选 `data-i18n-vars='{"n":3}'` JSON |
| `data-i18n-html` | `innerHTML`（慎用，需可信文案） |
| `data-i18n-title` / `data-i18n-aria-label` / `data-i18n-placeholder` | 对应 HTML 属性 |

动态插入的节点（如脚本生成的菜单卡片）在插入后应对其父级调用一次 `applyDom(container)`，并在后续语言切换时依赖全局 `applyDom(document.documentElement)`。

---

## 7. 业务接入要点

- **入口**：`web/src/main.js` 在 `DOMContentLoaded` 最先执行 `initI18n()` → `applyDom(document.documentElement)` → `applyMeta()`；监听 `#locale-select` 与 `subscribeLocale` 刷新 UI。
- **主题下拉**：`Game.refreshSkinSelectOptions()` 使用 `tSkinName()`；语言变更时需再次调用（已在 `subscribeLocale` 中挂载）。
- **每日大师题**：`web/src/social/dailyMaster.js` 使用 `t()` 与带 `data-i18n` 的菜单按钮。
- **样式**：主菜单浅色背景下语言下拉的高对比样式见 `web/public/styles/main.css` 中 `#menu .menu-locale-row`。

---

## 8. 新增一门语言（Checklist）

1. 复制 `web/src/i18n/locales/en.js`（或 `zh-CN.js`）为新文件，文件名 **`代码.js`**（例：`pt-BR.js`）。
2. 翻译全部键；占位符名必须与原文一致（如 `{{gap}}`、`{{seed}}`）。
3. 在 `i18n.js` 中：`import` 新模块，写入 **`LOCALES`**（键为语言代码字符串），并追加 **`AVAILABLE_LOCALES`**。
4. 运行 `npm test`（含 `tests/i18n.test.js`），确认无语法错误。
5. 本地切换语言检查：`lang`/`dir`、菜单、主题列表、Toast。
